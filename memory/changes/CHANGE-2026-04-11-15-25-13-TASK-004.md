## Detailed Changes
- Derived a new `UnitreeB2StairEnvCfg` from the existing B2 rough configuration and kept the original observation/action structure intact so the warm-start checkpoint remains shape-compatible.
- Added a local straight-stair terrain generator instead of reusing Isaac Lab's default pyramid stairs, because the requested v1 task needs a regular forward stair corridor aligned with the +x direction.
- Configured the stair terrain mix to use flat, simple rough, and straight stairs in a 20/20/60 split, and implemented three curriculum bands: low/wide stairs, denser stairs with light height perturbation, and longer stairs with light tread perturbation.
- Narrowed commands to forward stair-climbing behavior by disabling heading commands, fixing lateral velocity to zero, shrinking yaw rate, and fixing reset yaw to the stair direction.
- Tuned the inherited B2 rough rewards per the requested stair profile, preserved the explicitly requested weights, added stair-specific terms, and reused existing local/base Isaac Lab terms where appropriate (`bad_orientation`, `illegal_contact`, `undesired_contacts`).
- Added stair-specific termination logic for prolonged lack of forward progress and continuous backward slip, while reusing built-in `bad_orientation` and `illegal_contact` for tilt and body-collision exits.
- Added stair-specific runner/trainer config classes that only change `experiment_name` to `unitree_b2_stair`, leaving actor/critic architectures, observation dimensions, action dimensions, and checkpoint loading behavior unchanged.
