## 详细变更
- 调整 `StraightStairsTerrainCfg`，使其 `function` 属性直接在 `@configclass` 类体内声明，与 Isaac Lab 内建 terrain 配置模式保持一致，而不是在类创建后再赋值。
- 保留自定义楼梯 mesh 为明确的侧边支撑条、平坦起步平台、逐级抬升的台阶盒体和顶部平台，避免在楼梯下方放置整块重叠平面。
- 通过基于 marker 的 Isaac Lab smoke 脚本复现运行时阻塞，并把问题从 `gym.make(...)` 逐步缩小到 `TerrainGenerator(cfg)`，再进一步定位到自定义 terrain 配置上的 `copy()/to_dict()/function()` 交互。
- 在修复函数绑定方式后重新验证 terrain generator，并确认完整 stair 任务现在可以在真实 Isaac Lab 运行时 shell 中走通环境创建、reset、一次零动作 step 和环境关闭。

## 遇到的问题
- `RobotLab-Isaac-Velocity-Stair-Unitree-B2-v0` 的初始 Isaac Lab 运行时 smoke 在 `ENV_CREATED` 之前退出。基于 marker 的二分定位表明，失败发生在自定义楼梯地形生成阶段，而不是基础 B2 rough 任务。原因：`StraightStairsTerrainCfg.function` 在 `configclass` 声明之后才赋值，与 `TerrainGenerator` 使用的 `copy()/to_dict()` 路径不兼容。结果：通过把函数绑定移入 configclass 定义内部解决。
- 验证过程中出现 headless 运行时警告，包括 `GLFW initialization failed`、B2 足端的 USD visual reference 未解析，以及 Fabric point-instancer 警告。结果：这些警告在基线 B2 smoke 路径中也存在，但并未阻止本轮完成环境创建、reset、step 或正常关闭。
