## Detailed Changes
- 将 stair 任务的 `body_collision` termination 从检测 `base_link + hip + thigh + calf` 缩小为仅检测 `base_link + thigh`，并把终止阈值从 `25.0` 提高到 `60.0`，避免 shank 与 foot 的正常上阶接触被误判成重启条件。
- 保留 `body_collision_penalty` 奖励框架不变，仍对 `base_link`、`hip`、`thigh`、`calf` 的接触施加惩罚，但不再让 termination 过早打断刚出现的上阶行为。
- 将直楼梯 curriculum 从 3 档拉长到 5 档，前两档设为更保守的低台阶、宽踏面、短楼梯，并在最容易阶段关闭高度与踏面抖动，以便策略先稳定学会踩上第一阶。
- 调整直楼梯主配置：`num_rows=5`、`start_platform_length=2.0`、`top_platform_length=1.8`、`step_height_range=(0.04, 0.145)`、`step_depth_range=(0.60, 0.34)`、`num_steps_range=(3, 7)`、`height_jitter=0.004`、`depth_jitter=0.01`，让早期训练分布更宽容、难度提升更平滑。
- 对 stair 专用 PPO runner 下调探索强度：`policy.init_noise_std` 从 rough 继承值 `1.0` 降到 `0.6`，`algorithm.entropy_coef` 从 `0.01` 降到 `0.005`，`algorithm.learning_rate` 从 `1.0e-3` 降到 `7.5e-4`，以减少步态退化和无效摆动。
- 保持 actor/critic 输入输出维度、`joint_pos` 12 维动作、楼梯奖励框架和 rough checkpoint warm-start 兼容性不变。
